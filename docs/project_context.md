# ExcelToCSV 转换器 (项目记忆与设计文档)

## 1. 项目背景与目标
本项目旨在开发一个高性能的桌面端实用工具，专门解决业务中经常遇到的痛点：**大批量、超大型的 Excel 文件（.xls, .xlsx, .xlsm）在直接打开时极易导致系统资源耗尽或卡死**。
通过开发此工具，将这些重型表格批量、高效地按 Sheet 拆分并转换为轻量级的 CSV 纯文本文件，以便于后续的数据分析或导入处理。

## 2. 核心功能需求 (Feature Specifications)
1. **多文件批量导入**：支持同时选择或拖拽多个支持的 Excel 格式文件。
2. **Sheet 过滤机制**：
   - 提供配置项，允许用户输入 Sheet 名称的过滤清单（当Sheet名称中包含清单中的关键字时，不转换为CSV文件，默认不过滤）。
   - 默认功能：将一个 Excel 中的每个合法 Sheet 拆分为独立的 CSV。
3. **灵活的输出配置**：
   - **命名规则后缀选择**：
     - `[Excel文件名称]-[Sheet名称]_[时间戳].csv`
     - `[Sheet名称]_[时间戳].csv`
     - `[Excel文件名称]-[Sheet名称].csv`
   - **输出目录指定**：允许用户自定义转换后的 CSV 存放位置。
   - **文本编码选择**：支持下拉选择 `GBK`（针对旧版 Windows Excel 兼容）或 `UTF-8` 等常见编码。
4. **实时反馈监控**：
   - 清晰的已选文件列表及其单独的转换状态（等待中、转换中、成功、失败）。
   - 清晰的进度条,能直观看到每一个文件的进度，例如当前Excel文件有5个Sheet，处理到第二个Sheet时，进度条显示已完成2/5。
   - 实时的纯文本日志输出控制台，打印转换细节和错误信息。

## 3. 最终技术选型 (Tech Stack)
为了保证在处理 GB 级别文件时的内存稳定性和极速体验，同时兼顾开发效率和界面美观，严格遵循 KISS 和最小依赖原则，技术架构敲定如下：

### 3.1 前端 (Web UI)
*   **核心框架**: Vue 3 (Composition API)
*   **构建工具**: Vite
*   **开发语言**: TypeScript
*   **UI 框架**: Naive UI (选择理由：高度契合 TS、完全按需引入、极简极客的现代视觉风格，适合桌面工具)
*   **样式方案**: 原生 CSS (Vanilla CSS) 配合 Naive UI 基础主题，不引入额外 CSS 框架。

### 3.2 后端 (系统逻辑与运算核心)
*   **核心引擎**: Tauri (摒弃 Node.js 运行时，彻底解决内存溢出瓶颈，减小打包体积)
*   **开发语言**: Rust (无 GC 机制，内存释放精确，性能天花板)
*   **核心依赖库**:
    *   `calamine`: 业界顶尖的 Excel 解析库 (核心优势：同时完美支持遗留的 `.xls` 和现代的 `.xlsx/.xlsm` 格式)。
    *   `csv`: 高效的 CSV 构建写入器。
    *   `encoding_rs`: 解决转出 GBK 编码的核心库。

## 4. 架构工作流 (Workflow)
1.  **参数收集**: 用户在 Naive UI 构建的界面中选择文件并设置各项规则。
2.  **指令下发**: 前端通过 Tauri IPC (`invoke`) 将文件路径列表和配置参数 (JSON) 传递给 Rust 进程。
3.  **核心处理**: Rust 接管任务，利用 `calamine` 逐个加载文件，遍历 Sheets 执行过滤，并流式转换写入 CSV。
4.  **状态同步**: Rust 在处理期间，通过 Tauri Event 机制不断向前端抛出 `progress` 和 `log` 事件，前端响应式更新界面进度条和日志板。
