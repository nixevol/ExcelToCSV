# ExcelToCSV 转换器 (项目记忆与设计文档)

## 1. 项目背景与目标
本项目是一个桌面端 Excel 转 CSV 工具，用于将 Excel 文件（.xls, .xlsx, .xlsm）按 Sheet 拆分并转换为 CSV 文件，方便后续数据处理。

## 2. 核心功能需求 (Feature Specifications)
1. **多文件批量导入**：支持同时选择或拖拽多个支持的 Excel 格式文件。
2. **Sheet 过滤机制**：
   - 提供配置项，允许用户输入 Sheet 名称的过滤清单（当Sheet名称中包含清单中的关键字时，不转换为CSV文件，默认不过滤）。
   - 默认功能：将一个 Excel 中的每个合法 Sheet 拆分为独立的 CSV。
3. **灵活的输出配置**：
   - **命名规则后缀选择**：
     - `[Excel文件名称]-[Sheet名称]_[时间戳].csv`
     - `[Sheet名称]_[时间戳].csv`
     - `[Excel文件名称]-[Sheet名称].csv`
   - **输出目录指定**：允许用户自定义转换后的 CSV 存放位置。
   - **文本编码选择**：支持 `GBK` 或 `UTF-8` 编码。
4. **实时反馈监控**：
   - 清晰的已选文件列表及其单独的转换状态（等待中、转换中、成功、失败）。
   - 清晰的进度条,能直观看到每一个文件的进度，例如当前Excel文件有5个Sheet，处理到第二个Sheet时，进度条显示已完成2/5。
   - 实时的纯文本日志输出控制台，打印转换细节和错误信息。

## 3. 技术选型 (Tech Stack)

### 3.1 前端 (Web UI)
*   **核心框架**: Vue 3 (Composition API)
*   **构建工具**: Vite
*   **开发语言**: TypeScript
*   **UI 框架**: Naive UI
*   **国际化**: vue-i18n（语言包模式，支持中英文切换，便于扩展更多语言）
*   **样式方案**: 原生 CSS (Vanilla CSS) 配合 Naive UI 基础主题，不引入额外 CSS 框架。

### 3.2 后端 (系统逻辑)
*   **核心引擎**: Tauri v2
*   **开发语言**: Rust
*   **核心依赖库**:
    *   `calamine`: Excel 解析库，支持 `.xls`、`.xlsx`、`.xlsm` 格式。
    *   `csv`: 高效的 CSV 构建写入器。
    *   `encoding_rs`: GBK 编码支持。

## 4. 架构工作流 (Workflow)
1.  **参数收集**: 用户在 Naive UI 构建的界面中选择文件并设置各项规则。
2.  **指令下发**: 前端通过 Tauri IPC (`invoke`) 将文件路径列表和配置参数 (JSON) 传递给 Rust 进程。
3.  **核心处理**: Rust 利用 `calamine` 逐个加载文件，遍历 Sheets 执行过滤，并转换写入 CSV。
4.  **状态同步**: Rust 在处理期间通过 Tauri Event 向前端发送 `progress` 和 `log` 事件，前端实时更新进度条和日志。

## 5. 当前开发进度 (Progress)
- **2026-04-22**: 完成了界面 UI 布局重构与前端用户体验深度优化。
  - 前端 (`src/App.vue`): Vue 3 + Naive UI 搭建配置表单、文件上传、日志控制台与进度条界面。支持拖拽导入、可终止转换、主题切换功能。已补齐 9 种多语言支持（简体、繁体、英、日、韩、俄、法、西、德）。
  - 后端 (`src-tauri/src/lib.rs`): 使用 `calamine` 解析 Excel 并写入 CSV，支持通过 `AtomicBool` 标志响应前端的中断请求。
  - **界面交互优化**：重构了自适应伸缩策略，让文件列表区域（`top-card`）占满剩余屏幕高度，同时让底部状态面板（`bottom-card`）高度自适应，并限制日志框高度避免界面留白。增加了顶部工具栏按钮的 Hover 交互。
  - **应用环境安全**：通过 `onMounted` 拦截事件，禁用了全局右键菜单及 F12/Ctrl+Shift+I 等开发者控制台调出快捷键。
  - **配置持久化**：采用 `localStorage` 与 `watch` 自动保存机制，对界面主题、语言偏好、命名规则、字符编码、过滤关键字、最大线程数六项配置实现了持久化。
  - **转换性能优化**：CSV 写入改为 `BufWriter` 流式直写磁盘（UTF-8 零额外内存），GBK 模式通过自定义 `GbkEncoder` 流式编码；行缓冲区 `Vec<String>` 跨行复用减少内存分配。
  - **多线程并行转换**：引入 `rayon` 线程池，按文件维度并行处理，线程数可通过前端 UI 配置（1-32），默认 2 线程。每个线程独立持有 `row_buf` 和 `workbook`，通过 `AtomicBool` 共享取消信号。数字输入组件进行了样式强制覆盖，实现了精确居中对齐和分离按钮模式。
  - **CI/CD 发布自动化**：配置了 GitHub Actions (`.github/workflows/release.yml`)。支持通过打 `v*` 标签自动并发构建针对 Windows (x64/x86/ARM64), macOS (Apple Silicon/Intel), 和 Linux (x64/ARM64) 平台的原生可执行文件（含 AppImage 跨 Linux 发行版格式）并发布到 GitHub Releases 中。
- **2026-05-08**: 修复任务状态区域日志框滚动行为。
  - 前端 (`src/App.vue`): 底部状态卡不再拉伸占满剩余窗口高度，日志容器固定为 160px 高度，内容超出后在 Log 框内显示垂直滚动条，并保留日志更新后自动滚到底部；两个进度条禁用内置百分比指示器，避免开始转换后进度区高度变化把日志框顶出底部。
